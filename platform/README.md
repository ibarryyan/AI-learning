# 📘 Dify 简介

**Dify 是一个用于构建 AI 应用的可视化平台。**
它提供 LLM 调用、RAG、工作流编排、Agent 工具调用、应用发布等能力，开发者无需从零写代码即可快速搭建对话式应用、知识库问答、自动化任务执行等智能系统。

核心特点：

* 可视化 Workflow（流程编排）
* 内置 RAG / 向量库
* 支持 Agent 调用 HTTP/API 工具
* 多模型支持（OpenAI、Qwen、Kimi 等）
* 一键发布为 API 或 Web 应用

适合做：写作助手、企业问答、智能客服、自动化 AI 工具、复杂 Agent。

---

# 📗 n8n 简介

**n8n 是一款开源的自动化工作流工具（Workflow Automation）。**
它通过可视化节点连接各类服务（HTTP、数据库、GitHub、RSS 等），实现跨系统的自动化任务。
结合 LLM 节点后，n8n 可以执行智能判断、文本处理、内容生成等 AI 操作，使传统自动化具备智能能力。

核心特点：

* 可视化工作流编排
* 丰富的第三方服务节点（600+）
* 定时任务、监听任务、触发器丰富
* 可调用 LLM 作智能处理
* 自建/本地私有部署

适合做：数据自动化、内容推送、消息通知、信息监控、智能调度系统。

# 📊 Dify vs n8n 对比表（简洁易懂）

| 对比项        | **Dify**              | **n8n**           |
| ---------- | --------------------- | ----------------- |
| 工具定位       | AI 应用构建平台             | 自动化工作流平台          |
| 核心能力       | LLM、RAG、Agent、工作流     | 自动化任务编排、第三方集成     |
| 是否专为 AI 设计 | ✔ 是                   | ✖ 不是（但可结合 AI）     |
| 可视化程度      | 高：专为 AI 逻辑设计          | 高：专为自动化流设计        |
| 对 LLM 支持   | 原生支持、多模型              | 通过 LLM 节点支持       |
| 是否支持知识库    | ✔ 内置                  | ✖ 无（需外部工具）        |
| 是否支持工具调用   | ✔ 内置 Agent Tool + API | ✔ 通过 HTTP 节点/服务节点 |
| 适合场景       | AI 应用、智能问答、Agent、写作助手 | 自动化流程、数据同步、监控、通知  |
| 优势         | AI 原生、强 Agent 能力、快速开发 | 集成丰富、自动化强、触发器多    |
| 部署         | 开源、本地部署               | 开源、本地部署           |
| 使用难度       | Medium（需要理解 AI 逻辑）    | Easy（流程设计更直观）     |

# 🎯 最清晰的一句话总结

### **Dify = 用来构建 AI 应用（尤其是智能体、对话系统）**

### **n8n = 用来构建自动化流程（可以接入 AI）**

很多时候它们是互补的，而不是竞争关系。
